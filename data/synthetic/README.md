# Riverside Demo School District — synthetic dataset

**All data here is fictional. No real student data exists anywhere in this repository.**
Everything is generated deterministically (fixed seed, `uuid5` IDs, a pinned dataset
epoch) so regeneration is byte-stable and every harness metric is reproducible.

Generated by `scripts/build_dataset.py`; verified by `scripts/verify_dataset.py`
(schema validation + scripted findings + internal consistency + byte-stability). See
`PROVENANCE.md` for the full generation record.

## Layout

```
district/        School, 6 subjects, 8 teachers, 6 classes (1 co-taught), 12 students,
                 72 enrollments (1 mid-semester), Fall-2026 calendar incl. holidays.
                 district.json is the whole thing; the rest are split views for seeding.
ground_truth/    12 hand-authored canonical IEPRecords (<ref>.iep.json), each valid
                 against packages/schemas/IEPRecord.json (field_confidences embedded).
progress/        gradebook/<class>.csv, service_logs/<ref>.csv,
                 teacher_notes/teacher_notes.json, accommodation_confirmations.csv
documents/       Rendered IEP PDFs (STEP 4) and messy scans (STEP 5) — later slices.
```

## The reference date

The dashboard/deadline math evaluates against a fixed **`as_of` = 2026-11-13** (in
`district/calendar.json`), not the wall clock, so the demo and harness are reproducible
on any day. The Fall 2026 semester runs 2026-08-17 → 2026-12-18.

## field_confidences is embedded in the record

`IEPRecord.json` (schema v1.2) requires a top-level `field_confidences` object: six
0.0–1.0 scores for the canonical scalar/date fields (`student_ref`, `disability_category`,
`school_year`, `annual_review`, `triennial_reeval`, `last_progress_report`). A field whose
value is absent scores 0.0 — e.g. RIV-1012 has `last_progress_report: null` with confidence
0.0. Values are authored in `scripts/synthgen/ground_truth.py` and emitted inside each
record; there are no separate sidecar files.

## Source-grounded accommodation scopes

Every accommodation carries one or more `applies_to_refs` copied from document language.
The rendered form has a fixed four-page contract: accommodations on page 2, services on
page 3, and goals on page 4. Verification extracts each clean PDF text layer and proves
that every item and scope quote occurs verbatim on its declared page. RIV-1004 intentionally
keeps `all academic subjects` as one low-confidence subject phrase so it routes to review;
the generator does not invent a roster-derived subject expansion.

## The three scripted findings (STEP 2 — rules-engine acceptance fixtures)

Recomputed and asserted by `verify_dataset.py`:

1. **Goal-2 conflict** — RIV-1001 (A. Sharma): comprehension coursework averages **40%**
   (`gradebook/ENG-101.csv`, rows tagged `iep_goal_key=g2-comprehension`) while
   `teacher_notes.json` TN-0001 says "doing well".
2. **Extended-time gap** — RIV-1001 extended-time confirmed in **3 of 6** classes
   (`accommodation_confirmations.csv`): confirmed ENG/MTH/BIO, dropped HIS/PE/ART.
3. **Minute variance** — RIV-1002 (M. Bell) SAI delivered **130 min/wk vs 150 mandated**
   (`service_logs/RIV-1002.csv`): 26 min/session × 5 vs 30 min/session × 5 = −20 min/wk.

### Signal→goal mapping

Gradebook rows and teacher notes carry `iep_goal_key` + `iep_goal_id` annotation columns
giving the ground-truth signal→goal mapping (`goal_id = uuid5(namespace, "goal/<ref>/<key>")`).
Realistic assignment names are retained so a reconciler's own NLP mapping has real input.

### Noise (deliberate)

- **Unmappable signals**: gradebook rows with empty `iep_goal_key` (participation), and
  teacher note TN-0005 (no measurable goal).
- **Malformed CSV rows**: two rows (IDs `GB-9000x`) with wrong column count / non-numeric
  percent, appended to `gradebook/BIO-101.csv` and `MTH-101.csv` for the quarantine path.
- **Clean students**: RIV-1007…1012 have on-track data and no findings.

### accommodation_confirmations.csv

Stands in for teacher brief-receipt audit events until the briefs pipeline exists. One row
per (student, all-classes accommodation, class) with a `confirmed` flag. Feeds Family-4
gap detection.
